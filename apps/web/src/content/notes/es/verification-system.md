El sistema de verificación es el conjunto de mecanismos que
[Cocientífico](./coscientist) utiliza para contrastar [afirmaciones](./claim)
con evidencia, contraevidencia y explicaciones alternativas. No es una única
herramienta, sino un conjunto coordinado de protocolos y prácticas.

## Componentes principales

- [Verificación](./verification) — el proceso de contrastar afirmaciones
- [Búsqueda primero de refutación](./rebuttal-first-search) — buscar
  contraevidencia antes de aceptar
- [Protocolo de consenso multi-IA](./multi-ai-consensus-protocol) — usar el
  desacuerdo entre modelos como señal
- [Requisito de unanimidad](./unanimity-requirement) — aceptar afirmaciones solo
  cuando todos los revisores están de acuerdo
- [Línea de responsabilidad](./responsibility-line) — rastrear quién afirmó qué
- [Contraejemplo](./counterexample) — casos que invalidan afirmaciones
- [Ingesta de confianza cero](./zero-trust-ingestion) — exigir procedencia antes
  que autoridad

El sistema está diseñado para mantener que la
[corrección sea más barata que la deriva](./correction-vs-drift) y para
respaldar la [dificultad deseable](./desirable-difficulty-in-verification) de
modo que el [Operador](./operator) se mantenga involucrado.
