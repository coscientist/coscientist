"GitHub สำหรับนักวิทยาศาสตร์"
คือวิสัยทัศน์ที่มองว่าการทดลองทางวิทยาศาสตร์ควรถูกปฏิบัติเสมือนโค้ด:
มีการกำหนดเวอร์ชัน ทำซ้ำได้ และตรวจสอบได้
เช่นเดียวกับที่วิศวกรรมซอฟต์แวร์พัฒนาจากสคริปต์แบบเฉพาะกิจไปสู่การทำ Continuous
Integration และการควบคุมเวอร์ชัน
วิทยาศาสตร์ก็สามารถขยับจากสมุดบันทึกแล็บแบบครั้งต่อครั้งไปสู่ไปป์ไลน์ที่ทำซ้ำได้พร้อมการติดตามที่มา

วิสัยทัศน์นี้เป็นแกนกลางของ [ScienceOps](./scienceops)
เมื่อการทดลองกลายเป็นอาร์ติแฟกต์ที่มีเวอร์ชัน วงจรระหว่างสมมติฐานกับ
[การยืนยัน](./verification) จะสั้นลง ความล้มเหลวจะสามารถสืบย้อนรอยได้
ความสำเร็จจะสามารถทำซ้ำได้โดยผู้อื่นโดยไม่ต้องอาศัยความพยายามแบบฮีโร่

[นักวิทยาศาสตร์ร่วม](./coscientist) คือเลเยอร์ด้านความคิด
สำหรับเวิร์กโฟลว์ลักษณะนี้: จัดการ [ข้ออ้าง](./claim),
[หลักฐาน](./evidence-span) และข้อโต้แย้งย้อนกลับ ข้ามหลายโปรเจกต์
ขณะเดียวกันก็รักษา [เส้นความรับผิดชอบ](./responsibility-line) และสนับสนุนบทบาท
[วิศวกรวิทยาศาสตร์ธรรมชาติ](./natural-science-engineer)
