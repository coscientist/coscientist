O requisito de unanimidade é a regra no
[Protocolo de Consenso Multi-AI](./multi-ai-consensus-protocol) segundo a qual
[afirmações](./claim) são aceitas apenas quando todos os sistemas de IA
participantes concordam. A discordância entre modelos é tratada como um sinal
para inspeção mais detalhada, não como ruído a ser diluído por média.

Unanimidade não é prova de verdade; os modelos compartilham dados de treinamento
e podem convergir para falhas correlacionadas. Mas a discordância é um sinal de
baixo custo de que algo pode estar errado. O requisito de unanimidade força o
tratamento explícito da incerteza, em vez de permitir que uma maioria se
sobreponha a uma divergência.

A contrapartida é que a unanimidade pode empurrar a escrita para a cautela
excessiva e para conclusões do menor denominador comum. O protocolo compensa
exigindo que a incerteza permaneça explícita, em vez de ser suavizada em
linguagem vaga.
